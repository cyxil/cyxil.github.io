浅显易懂的Git+Github入门教程

发表于 2019-11-25 | 分类于 学习 | 阅读次数:

适合:还没有接触过Git+Github或刚刚接触还很懵逼的同学阅读

前言

准确的讲，这篇文章算不上是教程。

一是因为本人能力有限，二是因为这也许是你们看过的最简略的Git教程了。

那我为什么要写这篇文章呢？

在我们这一届，Git和Github是放在最后才讲的，也就是实习前，会讲一些基本操作，大概两节课的时间，之后要求自己练习。

当然，两节课的时间，足够把基础的命令讲完，实习前也可以熟练使用，但我觉得，不应该这么晚才去了解。

而是在学校期间，使用Github就应该成为一种习惯。至于原因，请看下文。

Git+Github是啥？能干吗？

设想一个场景，你有一个四人团队，要一起开发一个项目，每个人负责不同的模块，本来项目很顺利地进行着，但最后你要把所有人的代码合并，问题就来了。

每个人都改动了哪里，有没有删除或新增东西，如果合并错了怎么返回最初状态等等。

再设想一个场景，你写了一个很优秀的项目，你的室友想要来学习一下。一般，你会把项目压缩包发送给他，看上去没什么问题。

但过了一会，你发现项目有一个很严重的bug，修改完后你又重新打包发送给室友。又过了一会，室友发现你刚刚的改动出现了版本兼容问题，需要重写修改。一个室友还好说，多个人呢？

这些情况，会对我们的开发造成很大的影响。Git+Github 就解决了这个难题。

Github，代码托管平台，简单来讲，就是可以帮你储存项目，也可以让别人看到你的项目，储存代码的地方我们叫做仓库。

Git，分布式版本控制系统 ，它可以做到将本地代码上传到Github，也可以把别人的项目下载到本地，至于版本控制这篇文章就不多做描述了。

有了他们，上面的问题就很轻松了。

第一个场景，团队项目很难合并？每个成员完成代码后，上传到同一个仓库下，即使某一次提交出现了错误，也可以轻松的找出是谁上传了什么导致的。

第二个场景，我们把代码放到仓库中，让其他人自己下载使用，出了错也只需要改写代码再次提交，省下了很多步骤。

习惯使用Git+Github有什么好处？

提高代码水平

Github上有很多优秀的开源项目，可以供自己学习和参考。

在自己想写项目没有思路的时候，亦或是遇到BUG无法解决的时候，看一看大佬是怎么做的，对自己有很大的帮助。

见证自己的成长

最开始使用Github，我们并不需要有多好的项目，平时的练习，一小段Demo都可以放在上面，最好有一个可持续更新迭代的项目，比如你的博客，随着代码水平的提高，不断地完善，最终把它变成一个优秀的项目。

为找工作做准备

在找工作的时候，大部分公司都会在意你的项目经验，这个时候，如果你的Github上有很多优秀的项目，甚至还有一些Star，会起到很大的作用。
